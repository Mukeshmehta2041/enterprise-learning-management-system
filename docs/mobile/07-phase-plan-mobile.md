# 7. Mobile Phase Plan (Days 1–10)

This plan covers the **first 10 days** of LMS mobile implementation using React Native, aligned with backend and web frontend phases.

## Overview

| Day | Focus | Related Backend/Web Work |
|-----|--------|-------------------------|
| 1 | Mobile project setup & tooling | Backend Day 1, frontend Day 1 |
| 2 | Navigation shell and core screens | Frontend routing/layout docs |
| 3 | Mobile design system primitives | Web design system and components |
| 4 | Auth flows on mobile | Backend Day 2–3, frontend auth docs |
| 5 | Courses list & details screens | Backend Day 4, frontend courses UI |
| 6 | Content player & basic offline behavior | Backend content service, frontend player |
| 7 | Enrollments & progress on mobile | Backend Day 5, enrollment service |
| 8 | Assignments & submissions on mobile | Backend Day 8, assignments service |
| 9 | Notifications & basic payments on mobile | Backend Day 9, notification/payment services |
| 10 | Mobile polishing, performance, and release basics | Backend Day 10, frontend testing/observability |

## How to Use

1. Before each day, read the relevant mobile docs in `docs/mobile/` and cross-reference backend/frontend docs for APIs and behavior.
2. Open the corresponding mobile day task file under `tasks/` (e.g. `tasks/mobile-day-4.md`) and work through its checklist.
3. Mark progress in each day file as you complete items and keep notes on any follow-up work for later phases.
