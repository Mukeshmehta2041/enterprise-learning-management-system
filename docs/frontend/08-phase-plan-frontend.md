# 8. Frontend Phase Plan (Days 1–10)

This plan covers the **first 10 days** of LMS frontend implementation, aligned with the backend phases and daily tasks.

## Overview

| Day | Focus | Related Backend Days |
|-----|--------|----------------------|
| 1 | Frontend project skeleton & tooling | Backend Day 1 (monorepo structure) |
| 2 | Design system & layout shell | Backend Day 2–4 (core entities and APIs) |
| 3 | Auth flows & session management | Backend Day 2–3 (User, Auth services) |
| 4 | Course catalog listing | Backend Day 4 (Course service) |
| 5 | Course details & content player | Backend Day 4, 7 (Course, Content services) |
| 6 | Enrollment flows & progress views | Backend Day 5, 8 (Enrollment, Redis patterns) |
| 7 | Assignments & submissions UI | Backend Day 8 (Assignment service) |
| 8 | Notifications & payments UI | Backend Day 9 (Notification, Payment) |
| 9 | Analytics dashboards | Backend Day 9 (Analytics service) |
| 10 | Hardening: accessibility, performance, tests & CI | Backend Day 10 (Observability, resilience, CI/CD) |

## How to Use

1. Before each day, read the corresponding frontend doc(s) (e.g. Days 1–2 → `01-frontend-architecture.md`, `02-design-system.md`).
2. Cross-reference the backend docs and APIs for that day (e.g. Day 4 → `docs/api-specs/course-service-api.md`).
3. Open the matching **frontend day task file** under `tasks/` (e.g. `tasks/frontend-day-4.md`) and work through the checklist.
